# R1 verdict — Скептик (откат на z-ai/glm-5.2)

- роль: Скептик
- модель: z-ai/glm-5.2 (откат: deepseek/deepseek-v4-pro упал malformed после 1 retry)
- дата: 2026-07-05
- раунд: R1 (challenge, изолированный, fallback)
- пометка: пул 2/3, гетерогенность НЕ достигнута (glm ×2 + kimi)

---

<КРИТЕРИЙ 1 — Корректность>: VERDICT=FAIL — `skills/database.md` секция migration: «drizzle-kit generate up+down» — дриzzleльно неверно; `drizzle-kit generate` создаёт только forward SQL (файлы `*.sql`), down-миграцию нужно писать вручную или через отдельный инструмент. `agents/resilience.md` секция invariants: `wrap(CB, Retry, Bulkhead, Timeout)` — CB outermost означает, что каждая попытка retry-цикла считаетcя как CB-failure; при `retry_max=3` и `cb_fail_ratio` на 3 — один запрос с 3 транзиентными таймаутами открывает CB. `skills/database.md` секция транзакции: `db.transaction isolationLevel` — в Drizzle (postgres-js) API `db.transaction(async (tx) => {...}, {isolationLevel: 'serializable'})` не гарантирует поддержку isolationLevel как опции; может потребоваться `tx.execute(sql\`SET TRANSACTION ISOLATION LEVEL SERIALIZABLE\`)` внутри колбэка. <КРИТЕРИЙ 2 — Полнота>: VERDICT=PASS — все 10 секций контракта присутствуют во всех 6 skills; workflow domain→db→service→api→tests→docs описан; persistence N/A предусмотрен; stack binding без плейсхолдеров; gates реальные; bad/good ❌/✅; DoD; stop/escalation; audit trail. <КРИТЕРИЙ 3 — Реализуемость>: VERDICT=INCONCLUSIVE — gate `migration(up→down→up байт-в-байт)` (`skills/database.md` секция Gates) невыполним как описано, т.к. drizzle-kit не генерирует down-миграции автоматически; агент должен сначала написать down вручную, но файл не уточняет процедуру. `agents/vcs.md`: commit-msg-gate с тегом `[agent|assisted|manual]` требует кастомизации `type-enum` в commitlint config — стандартный `@commitlint/config-conventional` его не поддерживает. <КРИТЕРИЙ 4 — Риски>: VERDICT=FAIL — `agents/resilience.md`: нет total timeout budget across retries — при Timeout innermost суммарное время = `retry_max × timeout_ms`, что может превысить SLO без outer timeout; `skills/tests.md` gate `integration-fault-gate(timeout/5xx/429/dup-key)` не разделяет инструменты: toxiproxy инъецирует network-level сбои (timeout, connection reset), но НЕ HTTP 5xx/429 — для этого нужен undici MockAgent; `agents/resilience.md`: Bulkhead (cockatiel) + p-limit дублируют ограничение конкурентности без указания их scope-разделения; `skills/database.md`: down-миграция вручную → риск забыть DROP INDEX при expand-contract. <КРИТЕРИЙ 5 — Альтернативы>: VERDICT=INCONCLUSIVE — явное сравнение инструментов (cockatiel vs opossum, Drizzle vs Prisma, Vitest vs node:test) в сводке не видно; bad/good примеры показывают паттерны, но не tool-tradeoff. <КРИТЕРИЙ 6 — Соответствие контексту>: VERDICT=PASS — Node 24 LTS, ESM-native (Vitest, `*.mjs`), `npm ci`, `undici` bundled, `pino` structured logging, piscina worker_threads — все идиоматично для Node.js runtime. <КРИТЕРИЙ 7 — Паритет>: VERDICT=PASS — инварианты (cockatiel wrap, testcontainers, zod .strict(), onConflictDoNothing, keyset pagination, corrId) наблюдаются согласованно во всех файлах; cross-references между agents (resilience→performance, vcs→standard) консистентны.

ОБЩИЙ ВЕРДИКТ: NEEDS WORK

СПИСОК ПРАВОК:
1. `skills/database.md` секция migration: заменить «drizzle-kit generate up+down» на «drizzle-kit generate (forward-only) + ручное авторство down.sql»; добавить шаг «написать down.sql как обратную операцию к up.sql» в workflow.
2. `agents/resilience.md` секция invariants: изменить wrap order на `wrap(Timeout, Retry, CB, Bulkhead)` или `wrap(Retry, CB, Timeout, Bulkhead)` — CB должен отслеживать уникальные request-failures, а не per-retry failures; Retry должен обёртывать CB, чтобы retries могли пройти через half-open.
3. `agents/resilience.md` секция invariants: добавить total-timeout-budget (outer timeout = retry_max × per-attempt timeout + jitter margin) или явный `deadline` policy, предотвращающий суммарное превышение SLO.
4. `skills/database.md` секция транзакции: уточнить API — либо `db.transaction(async (tx) => { await tx.execute(sql\`SET TRANSACTION ISOLATION LEVEL SERIALIZABLE\`); ... })`, либо подтвердить, что Drizzle 0.36+ поддерживает `isolationLevel` в options и указать точный синтаксис.
5. `skills/tests.md` секция integration: явно разделить — toxiproxy → network-level (timeout, ECONNRESET, latency); undici MockAgent → HTTP-level (5xx, 429, malformed body); concurrent DB inserts → dup-key (testcontainers + 2 connections).
6. `agents/resilience.md` секция backpressure: уточнить scope — cockatiel Bulkhead для per-dependency concurrency (DB pool), p-limit для per-request fan-out; указать, что если используется только один, выбрать Bulkhead для внешних вызовов, p-limit для batch-операций.
7. `agents/vcs.md` секция commit-msg: добавить commitlint config с расширенным `type-enum` (или `scope-enum`), включающим `[agent|assisted|manual]` — стандартный `@commitlint/config-conventional` не покрывает кастомные теги.
8. `skills/database.md` секция Gates / migration: добавить gate-команду, которая проверяет существование и валидность `down.sql` (например, `scripts/migration-roundtrip.mjs` — apply up, apply down, apply up, compare schema hash).
