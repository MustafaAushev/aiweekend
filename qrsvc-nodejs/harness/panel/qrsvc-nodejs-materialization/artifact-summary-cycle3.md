### Изменения cycle-2 → cycle-3
- **[fix-2.1] cockatiel fullJitter**: `exponentialBackoff({jitter:true})` → `fullJitter(exponentialBackoff({baseDelay,maxDelay}))` (отдельная composable-обёртка; `import { fullJitter, exponentialBackoff } from 'cockatiel'`). TS strict принимает. Во всех файлах: resilience.md, feature.md, database.md, review.md.
- **[fix-2.2] cockatiel wrap order**: `Retry оборачивает CB` → **`CB оборачивает Retry`** (`wrap(circuitBreaker, wrap(retry, ...))`). CB видит финальный результат после retry, считает уникальные request-failures (одна операция = один CB decision). НЕ `wrap(retry, wrap(circuitBreaker))` — при Retry outermost каждая retry-попытка инкрементит CB counter → ложное OPEN. Обновлено: resilience.md (stack binding + wrap order + good-пример + DoD), feature.md (шаг 4), review.md (блок 2).
- **[fix-2.3] total deadline**: `AbortController`+`Date.now()` deadline = `retry_max × timeout_ms + jitter margin` (hard wall-clock для всей retry-цепочки; cockatiel `Timeout` per-execution ≠ total). В good-пример resilience.md — полный код с AbortController + signal.
- **[fix-2.4] Stryker incremental**: `--since main` (несуществующий флаг Stryker 8.x) → `git diff --name-only origin/main...HEAD -- 'src/**/*.ts' | xargs -r npx stryker run --mutate`. Во всех файлах: tests.md (шаг 8), test-matrix.md (T2 Eval), Makefile gate-mutation, standard.md §10.
- **[fix-2.5] migration-roundtrip pg_dump**: `pg_dump --schema-only --no-owner --no-acl --no-comments` для детерминированного hash (авто-gen constraint names). Makefile gate-migration.
- **[fix-2.6] zod 3.x pinned**: zod@^3 + @asteasolutions/zod-to-openapi@^7 (под zod 3.x). Для zod 4.x — `z.strictObject()`/`z.looseObject()` вместо `.strict()`. standard.md §10 таблица + §1.
- **[fix-2.7] gate-sizing**: голый grep → `node scripts/sizing-check.mjs` (allow-list, исключает комментарии/тесты). Makefile.
- **[fix-2.8] standard §1–§10**: ссылка обновлена (review.md, standard.md intro).
