# qrsvc-nodejs/harness/agents/fix-loop.md — правила авто-починки (падающий eval → агент чинит)
#
# Падающий тест = промпт: harness скармливает агенту traceback + diff, агент чинит до зелёного.
# Без ограничителей это бесконечный цикл, сжигание бюджета и «починка» через ослабление проверки.

## Non-negotiables
1. **F1 — Жёсткие лимиты (из `fix_loop.yaml`).** `max_iterations` ≤ 5 (hotfix ≤ 3); бюджет token/cost + wall-clock таймаут; превышение → эскалация человеку.
2. **F2 — Anti-cheat.** Агент не правит `immutable_paths` (harness, правила, пороги, eval-датасеты). Агент создаёт НОВЫЕ тесты, но НЕ правит существующие. Инварианты после фикса: зелёные тесты ≥ было; mutation score не упал; нет новых `skip`/`xfail`.
3. **F3 — Детект осцилляции.** Если каждая итерация ломает другой тест → стоп + эскалация.

## Quality gates (реальные команды)
- `grep -c 'max_iterations' fix_loop.yaml` — exists
- `grep -c 'immutable_paths' fix_loop.yaml` — exists
- `grep -c 'escalate_to_human' fix_loop.yaml` — on_exhausted

## Bad/good примеры
**Плохая починка (ослабление проверки):**
→ агент удалил падающий тест, чтобы CI стал зелёным.

**Хорошая починка (корневая причина):**
→ агент исправил логику, создал новый regression-тест, все старые тесты зелёные, mutation score не изменился.

## Stop rules
- Правка `immutable_paths` → RED FLAG (агент не владеет спекой)
- Удаление или `skip` существующего теста → stop
- Осцилляция (тест A починен → B упал → A снова упал) → stop + эскалация
- `max_iterations` исчерпан → эскалация, не «ещё раз»
