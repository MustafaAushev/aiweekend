# qrsvc-nodejs/harness/agents/incident-log.md — правила разбора инцидентов
#
# Агент не «учится» из одного фикса — повторит паттерн, пока нет абстрактного запрета.
# Каждый инцидент → правило → regression eval. Так операционный опыт превращается
# в архитектурные ограничения.

## Non-negotiables
1. **I1 — Формат записи инцидента обязателен.** Не «агент сделал баг», а извлечение аксиомы модели: симптом → аксиома → правило → regression.
2. **I2 — Инцидент замыкает петлю.** Каждый инцидент порождает: (1) изменение правила в `agents/*.md`, (2) новый regression eval-кейс, (3) при LLM-специфике — кейс в eval-датасет.
3. **I3 — Правило > локальный фикс.** Локальный фикс без правила запрещён — тот же класс отказа вернётся.

## Quality gates (реальные команды)
- `grep -c 'СИМПТОМ:' agents/incident-log.md` — число записей > 0
- `grep -c 'ПРАВИЛО:' agents/incident-log.md` — равно числу записей
- `grep -c 'REGRESSION:' agents/incident-log.md` — равно числу записей

## Bad/good примеры
**Плохая запись (нет правила, нет regression):**
```markdown
СИМПТОМ: 500 на POST /order
ФИКС: добавил try/catch
```
→ без правила — тот же баг вернётся в другом эндпоинте.

**Хорошая запись (полная петля):**
```markdown
СИМПТОМ:     p99 8с на GET /orders при 10k rows
АКСИОМА:     «OFFSET пагинация O(N) — это нормально для малых таблиц»
ПРАВИЛО:     Keyset pagination (`cursor`) обязателен на таблицах > 1k rows
REGRESSION:  EXPLAIN показывает Index Scan, не Seq Scan
```
→ правило добавлено в `agents/performance.md` (P1), regression eval в датасет.

## Stop rules
- Инцидент без аксиомы → не принят (симптом без корня = поверхностный фикс)
- Правка `agents/*.md` или `thresholds.yaml` без инцидента → stop (человек владеет правилами)
- Инцидент без regression eval → stop (петля не замкнута)
