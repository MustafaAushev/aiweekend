# rule: vcs — контроль версий и PR-дисциплина (язык-агностик)

> Единый источник git-правил. Скилы (feature/bugfix/hotfix/refactor/review/standard) ССЫЛАЮТСЯ сюда, не дублируют пороги. Пороги (размер PR и т.п.) — в `thresholds.yaml`. Плейсхолдеры `<VCS>`/`<DEFAULT_BRANCH>`/`<TICKET>`/`<LOCK_CMD>` — под свой стек.

## Нерушимые правила VCS

1. **Ветка на каждую задачу.** Работа только в ветке `<type>/<TICKET>-<slug>` (`feat`/`fix`/`hotfix`/`refactor`/`perf`/`chore`) от свежего `<DEFAULT_BRANCH>`. Прямой коммит/push в `<DEFAULT_BRANCH>` запрещён — она protected.
2. **Conventional Commits + тег вклада.** Каждый коммит: `[agent|assisted|manual] <type>(<scope>): <subject>`. Тег вклада обязателен (вход лидерборда части 4). Формат проверяется commit-lint'ом.
3. **Атомарность.** Один коммит = одно логическое изменение; репозиторий собирается и тесты зелёные на каждом коммите. Спека — отдельным коммитом РАНЬШЕ кода; спека+код+рефактор в одном коммите запрещены.
4. **PR-дисциплина.** Слияние только через PR: один PR = одна задача, diff ≤ `vcs.pr_max_lines` (`thresholds.yaml`), шаблон заполнен (Зачем/Что/Как проверить/Чеклист), ≥1 аппрув, автор себя не мёржит, merge лишь на зелёном CI + пройденном review-скиле.
5. **Хуки не обходить.** `--no-verify` / `SKIP=` запрещены на любой ветке, идущей в `<DEFAULT_BRANCH>`.
6. **Lock при зависимостях.** Тронул манифест зависимостей → пересобрал lock (`<LOCK_CMD>`, с хэшами) и закоммитил тем же PR.
7. **История неприкосновенна.** `push --force` в `<DEFAULT_BRANCH>`, перезапись тегов и чужой истории запрещены; fix-loop не трогает protected-пути.

**Eval (`vcs-gate`):** сообщения матчат `^\[(agent|assisted|manual)\] (feat|fix|refactor|perf|test|docs|build|ci|chore)(\(.+\))?: `; ветка ≠ `<DEFAULT_BRANCH>`; diff PR ≤ порога; при изменённом манифесте зависимостей hash-diff lock непустой; `git log --first-parent` не содержит прямых коммитов в `<DEFAULT_BRANCH>`. Любой пункт красный → merge заблокирован.

**Что ломается без правила:** агент коммитит прямо в `<DEFAULT_BRANCH>`, мешает спеку+код+рефактор в один коммит (невозможно ни отревьюить, ни откатить атомарно), обходит хуки `--no-verify` (красный код въезжает в main), меняет зависимости без пересборки lock (сборка невоспроизводима), force-push'ит историю. Лидерборд вклада `[agent]/[assisted]/[manual]` не собрать — коммиты без тегов.
